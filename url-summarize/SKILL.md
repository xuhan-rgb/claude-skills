# URL 内容总结 Skill

## 触发条件

用户给出 URL 并使用以下关键词时调用：
- "总结"
- "summarize"
- "解读"
- "/summarize"
- "/总结"

**可选：指定关注话题**
- "总结 xxx 关于YYY" → URL + 话题 "YYY"
- "summarize xxx 关注 ZZZ" → URL + 话题 "ZZZ"

**自动保存到文件**
- 默认行为：每次总结都会自动保存详细版 TXT 文件到当前目录
- 文件命名：`[页面标题]_summary_YYYYMMDD.txt`
- 如需只显示不保存，明确说明"仅显示"或"不保存"

## 执行流程

### 1. 提取 URL 和关注话题
从用户输入中识别：
- **URL**：http:// 或 https:// 开头的链接
- **话题关键词**：可选，用引号或"关于"、"关注"、"聚焦"等词标识

示例：
- "总结 https://example.com" → 仅获取页面总结
- "总结 https://example.com 关于 Python 异步" → 聚焦 Python 异步相关内容
- "summarize https://github.com/xxx 关注性能优化" → 聚焦性能优化相关内容

### 2. 获取网页内容
使用 WebFetch 工具获取页面内容：
```
WebFetch url="[提取的URL]"
```

### 3. 内容提取与总结

#### 如果指定了话题关键词：
1. 从页面全文中搜索包含话题关键词的段落/句子
2. 提取这些相关内容及其上下文
3. 围绕话题重新组织总结

#### 如果未指定话题：
提取以下关键信息：
- **页面标题**：从 HTML `<title>` 标签提取
- **主要内容**：从 `<article>`、`<main>`、`<div class="content">` 等主体标签提取
- **关键要点**：文章的核心观点、步骤、代码示例等

### 4. 内容增强（获取更详细信息）

在获取页面内容后，如果需要更详细的信息，可以进行额外处理：
- **代码示例**：提取所有代码块，保留完整格式
- **表格数据**：将 HTML 表格转换为 Markdown 表格
- **列表项**：保留所有有序/无序列表
- **标题层级**：保留 H1-H6 标题结构

### 5. 输出格式

**有指定话题时：**
```markdown
## 📄 页面总结（聚焦：[话题]）

**来源**: [URL]
**标题**: [页面标题]

### 话题相关的内容
[从页面中提取的与话题直接相关的内容]

### 相关要点
- 要点 1（与话题相关）
- 要点 2（与话题相关）
```

**无指定话题时：**
```markdown
## 📄 页面总结

**来源**: [URL]
**标题**: [页面标题]

### 核心内容
[1-3 句话概括页面主题]

### 关键要点
- 要点 1
- 要点 2
- 要点 3

### 相关链接/资源
[如有其他相关链接]
```

### 6. 自动保存到 TXT 文件

默认行为：每次总结都会自动保存详细版 TXT 文件。

1. 生成详细版总结（包含更多上下文和完整内容）
2. 使用 Write 工具保存为 TXT 文件到当前目录
3. 文件命名：`[页面标题]_summary_YYYYMMDD.txt`

**详细版 TXT 格式：**
```txt
================================================================================
页面标题
================================================================================
来源: [URL]
获取时间: [YYYY-MM-DD HH:mm:ss]

================================================================================
核心内容
================================================================================
[详细描述，2-3段话]

================================================================================
详细内容（完整提取）
================================================================================
[完整的主要内容，包含代码示例、表格等]

================================================================================
关键要点
================================================================================
- 要点 1: [详细说明]
- 要点 2: [详细说明]
- ...

================================================================================
相关链接
================================================================================
- 链接1: [URL]
- 链接2: [URL]

================================================================================
```

**触发方式：**
- "总结 xxx 保存到文件"
- "总结 xxx 保存"
- "summarize xxx save to file"

## 注意事项

- 如果页面是纯代码/GitHub 仓库，突出显示代码结构和用法
- 如果是文档页面，提取安装/使用步骤
- 如果是博客/文章，总结核心观点和结论
- 如果 WebFetch 失败，尝试使用 curl 命令获取
- 话题关键词支持多个词，用空格分隔表示"或"关系
