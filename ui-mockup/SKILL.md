# UI Mockup — 可视化设计原型生成

为手机 App、桌面 App 或 Web 页面生成高保真 HTML 交互原型，用浏览器预览后再落实到代码。

## 调用方式

```
/ui-mockup <描述>              # 为新功能/页面生成原型
/ui-mockup 审查                # 审查现有 UI 并生成优化原型
/ui-mockup 实现                # 将已确认的原型落实到代码
/ui-mockup 更新                # 根据最新代码更新 HTML 原型
```

## 工作流程

### Phase 1: 分析与设计（自动执行）

1. **理解上下文**
   - 读取项目的 CLAUDE.md、现有代码、框架类型
   - 确认目标：新建页面 or 优化现有 UI
   - 识别关键页面和状态（正常/空/错误/加载）

2. **识别平台类型**

   根据项目框架或用户指定，选择模拟框架：

   | 平台       | 框架类型识别                        | 模拟框架                           | 尺寸         |
   |------------|-------------------------------------|------------------------------------|--------------|
   | 手机 App   | Flutter / React Native / SwiftUI    | 手机模拟器（圆角+刘海+状态栏）     | 375×780px    |
   | 桌面 App   | Electron / Tauri / Qt / WPF         | 桌面窗口（标题栏+窗口控制按钮）    | 1200×800px   |
   | Web 页面   | React / Vue / Next.js / 传统 HTML   | 浏览器窗口（标签页+地址栏）        | 1440×900px   |

   用户可用 `/ui-mockup 手机 <描述>` 或 `/ui-mockup web <描述>` 强制指定平台。

3. **确定设计方向**
   - 根据应用类型选择基调：
     - **终端/开发工具类** → 深色终端风格（dark + mono font + 语义色）
     - **社交/内容类** → 轻量现代（明亮/圆角/层次感）
     - **工具/效率类** → 简约实用（高对比/大触摸区域）
     - **企业/数据类** → 结构化（表格/卡片/仪表盘）
   - 定义色彩系统：主色、语义色（成功/错误/警告）、背景层级（3-5级）
   - 选择字体策略：代码用等宽、UI用无衬线

4. **梳理页面与流程**
   - 列出需要可视化的屏幕/状态（通常 3-8 个）
   - 优先级排序：核心功能 > 交互状态 > 辅助页面
   - **识别交互流程**：哪些屏幕之间有导航关系

### Phase 2: 生成 HTML 原型

在项目目录下生成 `design_mockup.html`。

#### 通用格式要求

- 单文件自包含（所有 CSS 内联，无外部依赖，字体用 Google Fonts CDN）
- **按流程分组排列**，每条流程一行
- 屏幕下方标注编号和名称
- 真实内容（不用 lorem ipsum，用符合场景的中文/英文）
- 交互元素的视觉状态（选中/未选中/禁用）
- 如有弹窗/对话框，背景模糊遮罩

#### 手机 App 模拟框架

```
┌─────────────────────┐
│     ──────          │  ← 刘海/灵动岛
│  9:41    ■ ▊ 100%  │  ← 状态栏（时间、信号、电池）
│  ← 标题        ⋮   │  ← AppBar
│                     │
│   （页面内容区）     │  ← 375×780px
│                     │
│  🏠   🔍   👤      │  ← 底部导航栏（可选）
└─────────────────────┘
```

- 圆角边框 40px，刘海模拟居中
- 状态栏模拟（时间、电池图标）
- SafeArea 底部留白

#### 桌面 App 模拟框架

```
┌─────────────────────────────────────────────┐
│ 🔴 🟡 🟢  应用名称                    ─ □ ✕ │  ← 标题栏（macOS 左 / Windows 右）
├────────┬────────────────────────────────────┤
│        │                                    │
│ 侧边栏 │          主内容区                   │  ← 1200×800px
│        │                                    │
│ 导航项  │                                    │
│ 导航项  │                                    │
│        │                                    │
├────────┴────────────────────────────────────┤
│ 状态栏（可选）                               │
└─────────────────────────────────────────────┘
```

- 窗口圆角 12px，阴影 `0 8px 32px rgba(0,0,0,0.3)`
- 标题栏高度 38px，红黄绿按钮（macOS 风格）或 ─□✕（Windows 风格）
- 侧边栏宽度 220-260px，可折叠
- 支持菜单栏（文件/编辑/视图...），紧贴标题栏下方
- 快捷键标注：按钮旁用小标签显示 `⌘N` `Ctrl+S` 等

#### Web 页面模拟框架

```
┌─────────────────────────────────────────────────┐
│ ● 标签页名称    ✕ │  +                           │  ← 标签栏
│ ← → ↻  🔒 https://example.com/dashboard    ☆  │  ← 地址栏
├─────────────────────────────────────────────────┤
│                                                 │
│  Logo   导航1   导航2   导航3       🔔  头像    │  ← 顶部导航栏
│                                                 │
│ ┌──────┬───────────────────────────────────┐    │
│ │      │                                   │    │  ← 1440×900px
│ │侧边栏│         主内容区                   │    │
│ │      │                                   │    │
│ │      │                                   │    │
│ └──────┴───────────────────────────────────┘    │
│                                                 │
│  页脚：版权 · 关于 · 联系                        │  ← 页脚（可选）
└─────────────────────────────────────────────────┘
```

- 浏览器外壳圆角 12px，浅灰背景 `#DEE1E6`
- 标签栏高度 36px，地址栏高度 32px，含前进后退刷新按钮 + 锁图标
- 页面内导航栏高度 56-64px
- 主内容最大宽度 1200px 居中，两侧留白
- 侧边栏宽度 240px（如适用）
- **响应式标注**（可选）：同一页面展示 desktop + tablet + mobile 三个视口

#### 交互流程箭头（所有平台通用）

屏幕之间用**彩色箭头 + 文字**连接：

- **青色** — 普通导航/点击
- **紫色** — 选择特定类型
- **绿色** — 创建成功/保存完成
- **橙色** — FAB 或特殊操作

```html
<div class="flow-arrow purple">
  <div class="flow-arrow-text">选择 Claude</div>
  <div class="flow-arrow-line">
    <div class="flow-arrow-dash"></div>
    <div class="flow-arrow-head"></div>
  </div>
</div>
```

#### 可点击区域标注（所有平台通用）

- 用 **虚线边框 (dashed outline)** 高亮每个屏幕的核心交互元素
- 添加小标签说明点击后的效果（如 "点击进入 →"、"→ 屏幕 07"）
- CSS class: `.click-target` + `.click-hint`

#### 流程分组

- `<div class="section-title">` 标注每条流程名称
- FLOW A: 主流程 / FLOW B: 创建流程 / FLOW C: 空状态流程
- 流程内屏幕用 `<div class="flow-row">` 横向排列，箭头穿插其间

#### 顶部图例 + 底部说明

- **顶部图例 (Legend)**：箭头颜色、虚线框、状态指示点含义
- **底部设计说明**：改进点标注 NEW/CHG/保持，简述设计理由

#### 自动打开

- `xdg-open`（Linux）或 `open`（macOS）在浏览器中打开

### Phase 3: 确认与实现

用户查看原型后：

- **用户说"可以"/"开始"/"实现"** → 进入代码实现阶段
- **用户提出修改意见** → 更新 HTML 原型后再确认
- **用户说"只要原型"** → 结束

#### 代码实现策略（按平台）

**Flutter（手机/桌面）**：
1. 创建 `theme/app_theme.dart`，集中颜色/字体/间距
2. 从外向内：主题配置 → Widget 组件 → 页面样式
3. 用 `AppTheme.xxx` 替换硬编码值

**React / Vue / Next.js（Web）**：
1. 创建 CSS 变量或 theme 文件，集中定义 design tokens
2. 从布局到组件：Layout → 导航栏/侧边栏 → 页面内容
3. 响应式用 CSS Grid/Flexbox + media queries

**Electron / Tauri（桌面）**：
1. 窗口配置：标题栏样式、默认尺寸、最小尺寸
2. 前端框架同 Web 策略
3. 平台特性：系统托盘、原生菜单、快捷键绑定

## 设计原则

### 手机 App

- **触摸区域** ≥ 44×44pt，按钮最小 36px 高
- **信息层级**：标题 15-17px、正文 13-14px、辅助 11-12px
- **安全区域**：底部工具栏用 SafeArea 包裹
- **深色适配**：背景用 3-5 级灰度分层（不要纯黑 #000）

### 桌面 App

- **鼠标精度**：按钮可小至 28×28px，但间距 ≥ 4px
- **快捷键**：核心操作标注键盘快捷键
- **窗口缩放**：定义最小窗口尺寸（如 800×600），大屏自适应
- **拖拽区域**：标题栏可拖拽移动窗口（-webkit-app-region: drag）
- **多面板**：支持侧边栏折叠、面板拖拽调整宽度

### Web 页面

- **导航栏**：固定顶部，高度 56-64px，z-index 最高
- **内容宽度**：最大 1200px 居中，两侧 padding ≥ 16px
- **响应式断点**：mobile ≤ 768px / tablet ≤ 1024px / desktop > 1024px
- **表格/数据**：横向滚动 or 卡片切换（移动端）
- **加载状态**：骨架屏（Skeleton）优于旋转图标

### 配色模板

**终端暗色**（开发工具类）：
```
背景：#0A0E14 → #111720 → #151C28 → #1A2233 → #1E2940
文字：#E6EDF3（主）/ #8B9AB5（次）/ #546178（弱）
语义：绿 #3FB950 / 红 #F85149 / 橙 #D29922 / 蓝 #58A6FF / 紫 #BC8CFF / 青 #39D4E0
```

**明亮现代**（消费类/企业类）：
```
背景：#FFFFFF → #F8F9FA → #F1F3F5 → #E9ECEF
文字：#212529（主）/ #6C757D（次）/ #ADB5BD（弱）
主色：自定义品牌色 + 互补色
```

### 按钮风格

- **主要操作**：渐变填充（用 LinearGradient，不要纯色）
- **次要操作**：透明底 + 边框
- **危险操作**：红色文字或红底
- **快捷键按钮**：深色卡片 + 边框 + mono 字体

## 页面覆盖检查清单

生成原型前确认以下页面是否都已包含：

- [ ] **首页/列表页** — 主入口
- [ ] **表单页** — 新建/编辑
- [ ] **工作区/详情页** — 核心功能页面
- [ ] **弹窗/对话框** — 确认/输入/选择
- [ ] **空状态** — 首次使用/无数据时
- [ ] **菜单/下拉** — 操作选项展开态
- [ ] **错误/加载** — 异常状态（可选）
- [ ] **设置/偏好** — 配置页面（桌面/Web 常需）
